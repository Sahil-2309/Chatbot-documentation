Bull Queue — Why?
50 messages ek saath aaye toh?
Queue lagata hai — messages ek ek karke process hote hain (ya 3 parallel). Agar directly process karo aur Claude slow ho toh server hang hoga. Queue Redis-backed hai, fast hai, retry support hai.

02 — AI Engine — How It Actually Works
🧠
CRITICAL — AI Ko "Train" Nahi Karte
System Prompt = Agent ka Brain
Claude already trained hai billions of data pe. Tumhe sirf instructions deni hai har API call mein — isse System Prompt kehte hain. Yeh ek lambi text file hai (~2000-3000 words) jo define karti hai: agent kaun hai, kaise baat kare, kya rules hain, kaunse stages hain, response format kya hona chahiye. Yeh prompt har API call ke saath jaata hai. File change karo → behaviour change. No ML training needed.
System Prompt = Fixed instructions Sent every API call File: prompts/travel-agent.txt Change behaviour = edit file
Har API Call Mein Kya Jaata Hai
📄 SYSTEM PROMPT (Cached)
~2500 tokens • Same har baar • Prompt caching se 90% cost saved • Agent rules, tone, format, stages defined
💬 CONVERSATION HISTORY
Summary of old msgs (~200 tokens) + Last 10 messages (user/AI alternating) + Current stage from Redis
📨 NEW USER MESSAGE
Latest message from customer • ~50 tokens • Added as last item in messages array
▼ Claude processes all of the above and returns ▼

03 — Conversation Memory & Data Retention
Memory Layers — Kya Kahan Store Hota Hai
⚡
Redis — Hot Memory
Active session data. Current stage, conversation summary, mode (AI/human). Ultra fast reads.
TTL: 6 hours
After 6hrs no message → auto-expires
New message → fresh session created
🗄️
PostgreSQL — Permanent Memory
All conversations, customer profiles, leads data. Used to load last 10 messages for AI context.
Retention: 90 days
Cron job deletes older chats
Booking records: 1 year
🧠
Claude Context — Per Request
Summary of old msgs + last 10 messages sent each API call. Claude has NO memory between calls — you give it memory.
Window: ~10 messages + summary
Claude remembers NOTHING
YOU provide memory each time
Sliding Window Strategy
100 messages wali conversation mein bhi cost controlled
Message 1-90 ka AI-generated summary (~200 tokens) + Message 91-100 full text (~1500 tokens). Instead of 100 messages (20K tokens) → sirf 1700 tokens. Cost 12x kam.
// Har 10 messages pe summary regenerate
if (totalMessages > 10) {
  // Summarize messages 1 to (total-10)
  summary = await claude("Summarize this conversation...");
  await redis.set(`session:${phone}:summary`, summary);
}
24-Hour Window Rule (WhatsApp)
Meta ka strict rule — free reply sirf 24hr mein
Customer ka last message 24hr se purana → free-form reply nahi bhej sakte. Sirf pre-approved template message bhejo. Backend mein lastMessageAt track karo. Auto-detect: agar gap > 24hrs → template bhejo.
Templates: itinerary_ready, pricing_update, follow_up
if (now - lastMessageAt > 24 * 60 * 60 * 1000) {
  sendTemplate("pricing_update", { name, destination, price });
} else {
  sendFreeReply(aiResponse.reply);
}
04 — Human Agent ↔ AI Handover
Mode System — Kab AI, Kab Human
🤖
mode = "ai"
Default. AI processes all messages.
👨‍💼
mode = "human"
Human agent active. AI paused. Messages go to dashboard only.
🔄
Handback
Human clicks "Return to AI" → mode = "ai" again.
// Incoming message handler
const mode = await redis.get(`session:${phone}:mode`) || 'ai';

if (mode === 'ai') {
  // Normal flow — send to Claude, get response, reply
  await processWithAI(phone, message);
} else if (mode === 'human') {
  // Just save message, show on dashboard, NO AI call
  await saveAndNotifyDashboard(phone, message);
}

06 — Error Handling, Security & Safety Nets
💥
Claude API Down/Slow
Fallback reply + Queue retry
Pre-written fallback: "Abhi system thoda busy hai, 2 min mein reply karte hain." Bull queue mein retry — 3 attempts with 30s gap. After 3 fails → alert to admin + save message for manual reply.
🔐
Webhook Security
Signature verification mandatory
Meta sends X-Hub-Signature-256 header. Verify HMAC-SHA256 hash before processing. Reject unsigned/invalid requests with 403. Rate limit: same phone 10+ msgs in 1 min → temp block.
🔄
JSON Parse Failure
Claude ne format todha toh?
try-catch around JSON.parse. If fails → send Claude's raw text as reply, keep stage unchanged. Happens <1% of time. Better: use Anthropic tool_use for 99.9% guaranteed JSON.
🌐
WhatsApp API Failure
Reply send nahi hua toh?
Save reply in DB with status: "pending". Cron job every 1 minute retries pending messages. Max 5 retries. After that → mark failed + notify admin. Customer kabhi blank nahi rehna chahiye.
🧪
Testing Strategy
Real WhatsApp ke bina test karo
Meta provides test phone number for sandbox. NODE_ENV=development → mock all WhatsApp calls, log to console. Postman collection for API testing. Jest for unit tests on message processing logic.
📊
Analytics (Auto-collected)
Dashboard pe 4 key metrics
Response time (msg timestamp diff) • Conversion rate (confirmations ÷ total) • Drop-off (no reply > 7 days before confirm) • Popular destinations (extracted from AI intent). All from existing PostgreSQL data.
