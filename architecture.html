01 — Architecture Layers
📱
Layer 1
User + WhatsApp
Customer apna phone uthata hai aur WhatsApp pe message bhejta hai. Yahan se journey shuru hoti hai.
WhatsApp Business API Twilio / Meta Cloud API Webhook (HTTPS)
Input Types
Text messages, voice notes (transcribed), images (hotel/passport), quick reply buttons
⚙️
Layer 2
Backend Server
Webhook receive karta hai, message parse karta hai, session manage karta hai, aur AI ko forward karta hai.
Node.js + Express Twilio SDK Bull Queue (jobs) JWT Auth
Responsibilities
Rate limiting, session tracking, message queuing, spam pre-filter, webhook signature verification
🧠
Layer 3
AI / NLU Engine
Conversation ka dimaag. Intent samajhta hai, itinerary banata hai, spam detect karta hai.
Claude API (Anthropic) Prompt Templates LangChain (optional) Streaming Responses
AI Modules
Intent classifier · Itinerary generator · Spam filter · Feedback handler · Summarizer
🗄️
Layer 4
Data Layer
Conversations, sessions, itineraries, aur customer data — sab yahan store hota hai.
PostgreSQL (main DB) Redis (cache/session) Prisma ORM AWS S3 (files)
Data Stored
Chat history · Customer profiles · Itinerary drafts · Session state · Spam blocklist
👨‍💼
Layer 5 — Human Handover
CRM / Sales Dashboard
Jab itinerary confirm ho, AI ek clean summary CRM mein push karta hai. Human agent wahan pricing karta hai.
Custom Admin Dashboard WhatsApp re-entry API Email (Nodemailer) Notion / Airtable (optional)
☁️
Layer 6 — Infrastructure
Hosting + DevOps
Saara system yahan chalega. Scalable, secure, aur cost-effective setup.
AWS EC2 / Railway.app Nginx (reverse proxy) PM2 (process manager) GitHub Actions (CI/CD) Sentry (error tracking)

03 — Technology Deep Dive
📱
WhatsApp Layer
Customer touchpoint
  • Meta Cloud APIOfficial WhatsApp Business API — free tier available
  • Twilio (alt)Simpler setup, paid per message — good for MVP
  • Webhook HTTPSMessage aata hai toh Node.js ko POST request milta hai
  • Template MsgsPehla message sirf approved template se jaayega (Meta rule)
⚙️
Backend Server
System ka nerve center
  • Node.jsAsync I/O — WhatsApp jaise real-time apps ke liye perfect
  • Express.jsREST API routes, webhook handler, middleware chain
  • Bull + RedisMessage queue — agar AI slow hو toh queue mein daalo
  • ZodInput validation — malformed webhook data reject karo
🧠
AI / NLU Engine
Conversation intelligence
  • Claude APIMain LLM — intent, itinerary, summarization sab kuch
  • System PromptsAgent ka "brain" — rules, tone, guardrails pre-fed hote hain
  • Context WindowPuri conversation history AI ko dete hain — memory milti hai
  • Prompt TemplatesDiscovery / Itinerary / Filter / Handover — alag prompts
🗄️
Data Layer
Persistence + caching
  • PostgreSQLCustomers, conversations, itineraries — permanent store
  • RedisActive session state — fast access, TTL set karo
  • Prisma ORMType-safe DB queries — less bugs, easy migrations
  • AWS S3User photos, PDFs — file storage outside DB
👨‍💼
CRM / Dashboard
Human agent interface
  • Next.js (Admin)Sales team ka dashboard — incoming leads, itinerary view
  • WhatsApp re-entryAgent pricing type karta hai → customer ko auto-send
  • NodemailerEmail notification jab hot lead aata hai
  • Airtable (opt)Simple no-code CRM agar custom dashboard na banana ho
☁️
Infrastructure
Hosting + monitoring
  • Railway.appMVP ke liye — Node.js + Postgres + Redis sab ek jagah
  • AWS EC2Production scale ke liye — t3.medium kafi hai shuru mein
  • NginxReverse proxy + SSL termination + rate limiting
  • SentryError tracking — kab kahan crash hua pata chalega
